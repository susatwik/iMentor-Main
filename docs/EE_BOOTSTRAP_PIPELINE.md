# EE Bootstrap Pipeline

## Overview

The EE Bootstrap Pipeline converts the R24 Electrical Engineering syllabus PDF into structured data and prepares it for downstream ingestion by the iMentor RAG/Skill Tree system.

```
syllabus.pdf  ──►  syllabus.csv  ──►  syllabus_unified.csv  ──►  RAG Pipeline
 (119 pages)        (242 records)      (62 rows)                (Neo4j / Qdrant / Skill Tree)
```

---

## Architecture

### Node.js Pipeline (server/services/)

```
   bootstrapPipeline.js (step 1)
        │
        ├── pdfParserService.js       Parse PDF → extract 62 courses, modules, topics
        │                                Detect semesters (I-I through IV-II)
        │                                Extract prerequisites, outcomes, textbooks
        │
        ├── syllabusCsvGenerator.js   22-col CSV: Department...Remarks
        │                                One row per topic per course
        │
        └── syllabusValidator.js      Validate columns, structure, content
                                         (0 errors, warnings only)
```

```
   02_bootstrapping/ (step 2-3)
        │
        ├── convertEeCsvToUnified.js  22-col → 5-col unified format
        │                                (Module, Lecture Number, Lecture Topic, Subtopics, Resources)
        │
        └── bootstrapEeFull.js        Orchestrates full pipeline:
                                        1. PDF → CSV (bootstrapPipeline)
                                        2. Convert to unified CSV
                                        3. Generate keywords
                                        4. Print summary
```

### Python RAG Pipeline (server/rag_service/ — downstream)

```
   bootstrap_course.py (step 4+)
        │
        ├── POST /course/ingest       → curriculum_graph_handler → Neo4j
        ├── POST /pipeline/run/EE     → course_material_processor
        │       ├── Stage 8:  Study Questions
        │       ├── Stage 9:  Skill Tree  → skill_tree.json
        │       ├── Stage 10: Pedagogical Layers
        │       └── Stage 13: Lecture Notes
        └── generate_lecture.py       → lecture HTML/MD
```

## Execution Flow

### Command

```bash
# Run full EE bootstrap (step 1-3)
npm run bootstrap:ee

# Or equivalently:
node server/services/02_bootstrapping/bootstrapEeFull.js EE

# Individual steps:
npm run bootstrap:syllabus                 # Step 1: PDF → CSV
npm run bootstrap:ee:convert               # Step 2: 22-col → 5-col unified
npm run bootstrap:ee:keywords              # Step 3: generate keywords
```

### What happens

```
Step 1: PDF → CSV (862ms)
  62 courses parsed, 242 topic records
  Output: course_bootstrap/EE/syllabus.csv

Step 2: 22-col → 5-col unified (instant)
  62 unified rows
  Output: course_bootstrap/EE/syllabus_unified.csv

Step 3: Keywords (instant)
  TF-based keyword extraction per row
  Output: course_bootstrap/EE/syllabus_keywords_added.csv

Total: ~1 second
```

### CSV Column Mappings

**22-column format** (syllabus.csv):
`Department, Programme, Regulation, Semester, CourseCode, CourseTitle, Credits, Category, CourseType, Prerequisites, CourseOutcome, ModuleNumber, ModuleTitle, UnitNumber, UnitTitle, Topic, SubTopic, Keywords, TextBook, ReferenceBook, OnlineResource, Remarks`

**5-column unified format** (syllabus_unified.csv):
`Module, Lecture Number, Lecture Topic, Subtopics, Resources`

| Unified Column | Source (22-col) | Notes |
|---|---|---|
| Module | Semester | Maps to I-I, I-II, II-I, II-II, III-I, III-II, IV-I, IV-II |
| Lecture Number | auto-increment | Incremented per semester group |
| Lecture Topic | CourseCode + CourseTitle | e.g., "EE1011 - Basic Electrical Circuits" |
| Subtopics | Topic (comma-joined) | Deduplicated topic headers |
| Resources | TextBook; ReferenceBook; OnlineResource | Joined from first row of course |

## Troubleshooting

### Symptom: PDF parsing fails with "TT: undefined function: 21"

This is a benign warning from `pdf-parse`. It does not affect output. The parser extracts all 119 pages correctly.

### Symptom: Some courses have empty Semester

25 of 62 courses have empty Semester — typically electives and inter-departmental courses not listed in the main course listing tables. The RAG pipeline treats empty Module as ungrouped.

### Symptom: Keywords are empty (0 keywords)

The keyword generator's stopword list includes common EE terms like "circuit", "analysis", "system", "digital". This causes 0 keywords for most courses. To fix, update the stopwords in `keywordGenerator.js` or lower the `confidenceThreshold`.

### Symptom: CSV has fewer courses than expected

62 courses are extracted from the detailed syllabus pages. The R24 curriculum has ~105 unique course codes across all semesters + electives, but only 62 have full syllabus content (prerequisites, outcomes, textbooks) in the PDF.

## Future Departments (CSE/ECE/ME/etc.)

To bootstrap a new department:

1. Place `syllabus.pdf` in `server/course_bootstrap/<DEP>/`
2. Run: `node server/services/02_bootstrapping/bootstrapEeFull.js <DEP>`
3. *Note:* `pdfParserService.js` currently hard-codes the course code pattern `[A-Z]{2}\d{4}`. If the new department uses a different code format, update the regex.

### Template

```bash
# e.g., CSE
mkdir -p server/course_bootstrap/CSE
cp /path/to/CSE_syllabus.pdf server/course_bootstrap/CSE/syllabus.pdf
node server/services/02_bootstrapping/bootstrapEeFull.js CSE
# Verify output
head -3 server/course_bootstrap/CSE/syllabus.csv
```

### Known Regex Limitations

The parser assumes course codes match `[A-Z]{2}\d{4}` (2 uppercase letters + 4 digits). If a department uses:
- `CS101` (3 letters + 3 digits) — update regex
- `MExxx` (different prefix) — update stop list in `extractCourses` if needed
- Semester notation differs (e.g., `S1/S2` instead of `I-I/II-II`) — update `_extractSemesterMap`
